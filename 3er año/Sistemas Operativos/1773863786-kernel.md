### Ejecución Directa

En un sistema de ejecución directa, **los programas tienen acceso completo a todo el hardware del sistema**, sin ningún tipo de control.

El sistema operativo, provee rutinas a modo de biblioteca. Actúa principalmente como *pegamento*, pero no *provee aislamiento*.


> [!Danger] Problema
> La ejecución directa no provee aislamiento y protección.

Ejemplos:
- Nada garantiza que un programa no modifique a otro programa (o al SO mismo).
- Acceso directo al storage permanente: nada garantiza que un programa no dañe el file system.
- Nada garantiza que un programa termine, se interrumpa, etc.

### Ejecución directa limitada

1. El *hardware* va a limitar ciertas operaciones.
2. Un programa especial y privilegiado va a arbitrar las operaciones riesgosas.

> [!Quote] "El rol central de un sistema operativo es la protección." - Fernando J. Corbató

### User-space vs Kernel space

#### [[User space]]
- Una **aplicación** puede ejecutarse **solo en modo usuario** (modo restringido).
- No puede ejecutar instrucciones privilegiadas.
- Se dice que se ejecuta en **user space**.

#### Kernel space
- **Solo el kernel** se ejecuta en modo supervisor.
- Puede ejecutar instrucciones privilegiadas.
- Se dice que se ejecuta en **kernel space**.

**El hardware** controla quien, *en ultima instancia*, controla que los programas (user-mode) se comporten correctamente.
- El hardware provee el **mecanismo de protección**.
- El software (kernel) determine la **política de protección** y configura el hardware.

### System calls
- Una *system call* **cambia el modo del procesador de user mode a kernel mode**.
- El **conjunto de system calls es fijo**. Cada syscall está identificada por un único número no visible al programa.
- Cada syscall debe tener **un conjunto de parámetros** que especifican información que debe ser transferida desde el user space al kernel space.

### Instrucciones privilegiadas

> [!Info]- Definición informal
> Toda instrucción que el modo usuario no puede ejecutar.

#### Modo protegido en x86
Para pasar de Modo Usuario (CPL 3) a Modo Supervisor (CPL 0) mediante una syscall hay un protocolo bien definido que los programas ejecutan.

#### Ejemplos de instrucciones privilegiadas
- *MOV* a/desde CR0-CR4 (Control Registers): Movimientos de datos hacia o desde los registros de control CR0, CR2, CR3, CR4. Estos registros controlan aspectos críticos del procesador.
- *HLT (halt)*: Detiene la ejecución del procesador hasta que ocurra una interrupción. Utilizada típicamente para poner al procesador en un estado de espera.
- *LGDT (Load Global Descriptor Table)*: Carga la dirección de la GDT en el registro GDTR, crucial para la administración de memoria y protección de segmentos en SOs que utilizan la memoria segmentada.

### Timer Interrupts (Interrupciones por temporizador)
Casi todos los procesadores contienen un dispositivo llamado **Hardware Timer**, cada timer **interrumpe** a un determinado procesador mediante una interrupción por hardware.
Cuando una interrupción por tiempo se dispara se transfiere el control desde el proceso de usuario al Kernel.
