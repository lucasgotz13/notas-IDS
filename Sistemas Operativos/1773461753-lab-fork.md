# Lab fork

## Introducción: ping-pong
- Dos procesos: padre e hijo
- Proceso padre envía un mensaje al proceso hijo
- Proceso hijo recibe y contesta
- Ambos procesos terminan

¿Cómo creamos los procesos? => `fork()`
### Fork(2)
- Crea un nuevo proceso
- Idéntico en **todo sentido** al proceso que llama a fork
- **Excepto** por el valor que **retorna fork**
- Puede fallar!
#### Ejemplo
```c
int main(int argc, char* argv[]) {

  int a = 4;

  int i = fork();

  a = 5;

  if (i < 0) {
    printf("Error en fork! %d\n", i);
    exit(-1);
  }
  
  if (i == 0) {
    printf("[hijo] mi pid es: %d\n", getpid());
    printf("[hijo] a=%d\n", a);
  } else {
    a = 6;
    printf("[padre] mi pid es: %d\n", getpid());
    printf("[padre] a=%d\n", a);
  }
  
  printf("Terminando\n");
  exit(0);
}
```

## Resolución ping-pong

```c
#include <sys/types.h>
#include <unistd.h>

int main()
{
	
	int fds_padre_hijo[2]; // [read, write]
	int fds_hijo_padre[2];
	
	int msg = 42; // es el "random"
	
	pipe(fds_padre_hijo);
	pipe(fds_hijo_padre);
	
	// tabla fds hasta el momento
	// 0: stdin
	// 1: stdout
	// 2: stderr
	// 3: pipe1_read
	// 4: pipe1_write
	// 5: pipe2_read
	// 6: pipe2_write
	
	pid_t p = fork();
	
	READ_FD = 0;
	WRITE_FD = 1;
	
	if (p == 0) {
		// hijo
		close(fds_padre_hijo[WRITE_FD]); // No se usan en este proceso
		close(fds_hijo_padre[READ_FD]); // No se usan en este proceso
		
		int buf_lectura;
		
		read(fds_padre_hijo[READ_FD], &buf_lectura, sizeof(buf_lectura));
		
		printf("Hijo recibe valor: %d\n", buf_lectura);
		
		write(fds_hijo_padre[WRITE_FD], &buf_lectura, sizeof(buf_lectura));
		
		close(fds_padre_hijo[READ_FD]); 
		close(fds_hijo_padre[WRITE_FD]);
	} else {
		// padre	
		close(fds_padre_hijo[READ_FD]); // No se usan en este proceso
		close(fds_hijo_padre[WRITE_FD]); // No se usan en este proceso
		
		int buf_lectura;
		
		write(fds_padre_hijo[WRITE_FD], &msg, sizeof(msg));
		read(fds_hijo_padre[READ_FD], &buf_lectura, sizeof(buf_lectura));
		printf("Padre recibe valor: %d\n", buf_lectura);
		
		close(fds_padre_hijo[WRITE_FD]); 
		close(fds_hijo_padre[READ_FD]);
		
		wait(NULL);
	}
	
	return 0;
}
```
### Comentarios:
- Tienen que ser 2 pipes: uno para conectar el padre con el hijo (ping) y el otro para conectar el hijo con el padre (pong).
- [[Tabla File Descriptors General]]
- El `close()` es propio del file descriptor siempre.
- Es importante cerrar los fd cuando se pueda ya que si por ejemplo hacemos otro fork(), los fd se van a duplicar.
