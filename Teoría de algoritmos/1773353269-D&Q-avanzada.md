## Problemas avanzados de División y Conquista
### Problema 1: Multiplicación de números **muy** grandes
Cuando multiplicamos dos enteros de largos m y n, la complejidad es O(mxn).
Hay alguna forma de hacerlo mejor?
#### Solución
Escribimos la multiplicación como si estuviéramos trabajando en base 2, y separamos en primera mitad y segunda mitad: 
![[Pasted image 20260312193855.png]]
Sigue siendo cuadrático => Solución: **Algoritmo Karatsuba-Offman**
```
def multiplicaciónBigInt(x, y):

si largo de x e y son pequeños, devolvemos x*y, sino:

x = x1 2n/2 + x0

y = y1 2n/2 + y0

p = multiplicaciónBigInt(x1 + x0, y1 + y0)

x0y0 = multiplicaciónBigInt(x0, y0)

x1y1 = multiplicaciónBigInt(x1, y1)

return x1y1 2n + (p - x1y1 - x0y0) 2n/2 + x0y0**
```
### Problema 2: Obtener extremo de un polígono
Tenemos n vértices V = (V0, V1, ... Vn-1), dispuestos en sentido anti-horario.
Problema: obtener el vértice extremo respecto a un eje (u).
Solución sencilla => Ver todos los vértices => O(n)
**Se puede hacer mejor** siempre y cuando el polígono sea convexo:
- Que para todo segmento L el polígono sea Monotónico
- Que no lo corte más de 2 veces
- Equivalente a que los ángulos interiores <= 180°
#### Solución
Lo mas importante es "el sentido respecto a la proyección" que tiene cada vector
Supongamos que el máximo está entre los vértices A y B, vemos en el dibujo cómo tenemos el segmento A,B  y obtenemos el vértice C, del medio.
Algoritmos:
- Empezamos con A = 0, B = n-1 y C = n/2.
- Si tenemos 2 vértices/puntos, comparemos a mano.
- Aplicamos la lógica de partir.
- Como partimos a la mitad, nos quedamos con una sola, y luego sólo comparo => símil búsqueda binaria => $T(n) = T(n/2) + O(1) => T(n) = O(\log n)$.
### Problema 3: Buscando puntos más cercanos en 2 dimensiones
Dado n puntos en un plano, buscar la pareja que se encuentre más cercana.
Algoritmo sencillo: ver todas las distancias => $O(n^2)$
#### Solución
Asumimos que ningún par de puntos tienen misma coordenada x o y.
¿Cómo resolveríamos el problema en una dimensión? => **ordenamos** => $O(n\log n)$
Igualmente no resuelve el problema en 2 dimensiones, pero se puede usar nociones de Mergesort para resolverlo.
Vamos a buscar la pareja más cercana del lado izquierdo, luego otra del lado derecho, y luego en tiempo lineal buscar los más cercanos => $O(n \log n)$
(Ese último paso lineal no es sencillo de definir)
```
def closest_pairs(P):
	Construimos px y py (O(n log n))
	p0, p1 = closest_pairs_rec(px, py)
	return p0, p1
	
def closest_pairs_rec(px, py):

	if len(px) <= 3: return el mínimo de comparar cada punto

	Construir Qx, Qy, Rx, Ry (O(n))

	q0, q1 = closest_pairs_rec(Qx, Qy)

	r0, r1 = closest_pairs_rec(Rx, Ry)

	d = min(dist(q0, q1), dist(r0, r1))

	x* = máxima coordenada x de Qx

	S = puntos de P que están a distancia <= d de la recta x = x*

	Construir Sy (O(n))

	por cada punto s de Sy computar distancia contra los siguientes 15 puntos

		quedarse con s y s' que minimizan esa distancia

	if dist(s, s') < d: return s, s'

	elif dist(q0, q1) < dist(r0, r1): return q0, q1

	else: return r0, r1
```
### Problema 4: Multiplicación de matrices
Primera idea: Dividir cada matriz de n x n en 4 submatrices de n/2 x n/2 => sigue siendo $O(n^3)$
